考試重點在哪？

這題在考你能不能將演算法分為兩個等級：

  • 慢郎中等級 O(n²)：資料變多，時間暴增。
  • 資優生等級 O(n log n)：資料變多，時間增加得比較合理。

選項逐一破解

(A) Bubble Sort (氣泡排序) ▼ 點擊看解析

錯誤

就像氣泡從水底慢慢浮上來，它每次只能交換相鄰的兩個數字。

平均複雜度：O(n²)

評語：資料一多就慢到懷疑人生。

(B) Selection Sort (選擇排序) ▼ 點擊看解析

錯誤

每次從還沒排好的堆裡「選擇」一個最小的放到前面。

平均複雜度：O(n²)

評語：不管資料原本亂不亂，它都要完整跑完兩層迴圈，很死腦筋。

(C) Merge Sort (合併排序) ▼ 點擊看解析

正確

採用Divide and Conquer (分治法)。先把資料切一半、再切一半...直到剩下一個，排好後再慢慢「合併」起來。

平均複雜度：O(n log n)

評語：切分是 log n 層，每層合併花 n，乘起來就是 n log n。穩定且快速！

(D) Insertion Sort (插入排序) ▼ 點擊看解析

錯誤

像玩撲克牌一樣，把拿到的新牌「插入」到手中已排好的位置。

平均複雜度：O(n²)

評語：雖然在資料「幾乎排好」時很快，但平均來說還是慢。

2. 資料量很大時，O(n²) 與 O(n log n) 誰比較快？

3. Bubble, Selection, Insertion 這三兄弟平均是哪個等級？

名師口訣與陷阱

【記憶口訣】

「平方三兄弟，合併最聰明」

  • 平方三兄弟 (O(n²))： Bubble, Selection, Insertion
  • 合併最聰明 (O(n log n))： Merge Sort

【常見陷阱】

  1. 只背英文名字，不背複雜度。
  2. 看到 Sort 就亂選 Bubble (因為名字最好記)。
  3. Quick Sort (快速排序) 平均也是 O(n log n)，但最差會變成 O(n²)，要小心題目問什麼！(本題未列出 Quick Sort，Merge Sort 最穩)。
C

Merge Sort (合併排序)

理由：利用分治法與遞迴，平均複雜度為 O(n log n)。
