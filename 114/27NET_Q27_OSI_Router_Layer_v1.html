選項詳細解析

(A) 資料連結層 顯示解析
✕ 錯誤
這一層主要是 Switch (交換器) 在用的，它看的是 MAC 位址 (實體位址)，只管區網內的傳送。
(B) 網路層 顯示解析
○ 正確！
Router 的核心功能是「路由 (Routing)」，也就是根據 IP 位址幫封包找最佳路徑，這正是網路層 (Network Layer) 的定義。
(C) 傳輸層 顯示解析
✕ 錯誤
這一層負責端到端的連線品質與流量控制 (如 TCP / UDP 協定)，不是 Router 轉送封包的主要層級。
(D) 應用層 顯示解析
✕ 錯誤
這是最上層，直接接觸使用者與應用程式 (如 HTTP, FTP, Email)，不是網路設備的工作層。

口訣與常見陷阱

【記憶口訣】

「找路靠 IP，Router 在第三」

（第三層 ＝ 網路層）

  • 陷阱 1：把 Router (L3) 和 Switch (L2) 層級搞混。記住：Switch 只有 MAC，Router 才有 IP。
  • 陷阱 2：看到「封包」就亂選傳輸層。傳輸層管的是「連線」，網路層管的是「路徑」。
  • 陷阱 3：OSI 七層順序不熟。請由下往上背：實體、資結、網路、傳輸...。
答案：(B)

一句話理由：Router 依 IP 位址決定封包路徑，屬於網路層 (Network Layer) 的核心功能。
