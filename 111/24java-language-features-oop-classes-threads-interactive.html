題目 24：Java 程式語言特性

高中物件導向入門 • 互動式講義

題目 24：關於 Java 程式語言之敘述，下列何者正確？

  • (A) Java 支援如同 C++ 程式語言的多重繼承
  • (B) 在 Java 中，可直接寫程序，而不用定義任何類別
  • (C) 在 Java 中，一個應用程式是一堆類別與這些類別實例的集合
  • (D) Java 程式以直譯器（interpreter）執行，只允許單一執行緒
正確答案：(C)

觀念暖身：專有名詞白話文

類別 (Class) 像是「餅乾模具」或「設計圖」。定義了東西長什麼樣子、有什麼功能。
物件/實例 (Object/Instance) 像是用模具壓出來的「真餅乾」。依照類別產生出來的實際東西。
多重繼承 (Multiple Inheritance) 一個小孩有好幾個親生爸爸。Java 的類別不支援這個（太亂了），但 C++ 可以。
介面 (Interface) 像是「契約」或「規格書」。只規定你要做什麼功能，不規定怎麼做。
執行緒 (Thread) 工廠裡的生產線。Java 支援多執行緒，也就是可以同時開好多條生產線一起工作。

觀念一：為什麼一定要有 Class？

請點擊下方程式碼的不同區塊，看看它們代表什麼意義：

class App { public static void main(String[] args) { System.out.println("Hello Java!"); } }
點擊上方程式碼有顏色的文字，這裡會顯示說明喔！

觀念二：選項逐一解析

點擊下方選項卡片，查看為什麼對、為什麼錯：

❌ 錯誤！
Java 為了避免混亂（例如：鑽石繼承問題），規定類別只能單一繼承（一個小孩只能有一個親生爸爸）。
若需要類似功能，Java 使用「介面 (Interface)」來達成（可以實作多個介面）。
❌ 錯誤！
Java 是強制的物件導向語言。所有的程式碼、函式（方法），都必須被包在一個 class (類別) 裡面。
不像 C 或 Python 可以直接寫在外面。
⭕ 正確！
這就是物件導向 (OOP) 的精髓！
Java 程式運作時，就是由許多設計圖（類別）產生出許多物件（實例），然後這些物件互相溝通合作。
❌ 錯誤！這句話錯了兩個地方：
1. 執行方式：Java 是先編譯成 .class (Bytecode)，再由 JVM 執行 (包含直譯與 JIT 編譯)，不是純直譯。
2. 執行緒：Java 非常強調支援多執行緒 (Multi-thread)，可以同時做很多事！

觀念三：繼承 vs 介面 (考試必考)

很多同學搞混，請看這個對照表（請記住關鍵字）：

特性 類別繼承 (extends) 介面實作 (implements)
數量限制 只能 1 個 (單一繼承) 可以 無限多個
目的 沿用父類別的功能 遵守某種規格契約
口訣 爸爸只能有一個 證照可以考很多張

觀念四：執行與執行緒

點擊卡片翻開真相：

Java 怎麼執行？

原始碼(.java) ➔ 編譯 ➔ 位元碼(.class) ➔ JVM (虛擬機) 執行
(並非單純直譯，效率較高！)

單一 vs 多執行緒？

Java 支援「多執行緒 (Multi-thread)」！
(就像餐廳裡有好幾個服務生同時服務，不會只有一個人忙死)

⚠️ 考試常見陷阱區

這些觀念要是搞錯，分數就送人了！點擊「我懂了」把錯誤觀念消除。

陷阱 1：「Java 跟 C++ 一樣有多重繼承」
錯！Java 類別只有單一繼承。要多重功能請找 Interface。
陷阱 2：「Java 不用寫 class 就能跑」
大錯特錯！Java 是強制物件導向，連最簡單的 Hello World 都要包在 class 裡。
陷阱 3：「Java 只能單執行緒，跑很慢」
錯！Java 強項就是多執行緒伺服器端應用，且有 JVM 優化，速度不慢。

Q3：多執行緒 (Multi-thread) 對應用程式有什麼好處？
提示：不要讓程式「卡住」。可以同時做 A 事又做 B 事 (例如邊下載邊看影片)。
