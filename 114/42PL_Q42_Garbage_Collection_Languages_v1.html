  • (E) JavaScript

【白話翻譯】這題到底在問什麼？

題目翻譯：
「哪些程式語言在寫程式時，不需要工程師自己寫程式碼去釋放記憶體，而是交給系統（虛擬機或執行環境）自動在背景處理？」

這題在考什麼：
你是否能分清楚哪些語言屬於「自動記憶體管理（有 GC）」，哪些屬於「手動記憶體管理（無 GC）」。

記憶體管理大對決（點擊語言看原因）

自動管理 (有 GC)
  • Java
  • Go
  • JavaScript
手動管理 (無 GC)
  • C
  • C++
【選項詳解】為什麼選這些？
正確 (A) Java
JVM (Java 虛擬機) 最著名的功能就是強大的垃圾回收機制。
正確 (B) Go
Go 語言是現代語言，Runtime 內建高效能的 GC，減輕開發者負擔。
錯誤 (C) C
C 語言最底層，完全信任工程師，記憶體若不手動 free 掉，就會一直佔用。
錯誤 (D) C++
C++ 為了效能與相容 C，主要依賴解構子 (Destructor) 與手動管理，標準庫無內建全自動 GC。
正確 (E) JavaScript
不論是 Chrome (V8 引擎) 還是其他環境，JS 都是自動回收記憶體的。

現在換你選選看（複選題）

正確答案：A、B、E

一句話總結：
「Java、Go、JavaScript 皆由系統自動進行垃圾回收，C/C++ 則需自行管理。」

★ 補教名師口訣 ★

「J G J，有 GC」

（Java、Go、JavaScript）

⚠️ 常見陷阱注意
  • 以為 C++ 有 GC（其實它是靠 RAII 機制，不是背景 GC）。
  • 以為 JavaScript 很簡單所以沒有記憶體管理（其實是因為它幫你做掉了）。
  • 看到「指標 (Pointer)」就以為要手動管理（Go 也有指標，但 Go 有 GC）。
