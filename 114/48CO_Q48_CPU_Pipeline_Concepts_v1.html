48. 關於「指令管線（Pipeline）」的敘述，哪些是正確的？（複選）

(A) Pipeline 可以提高指令吞吐量
(B) Pipeline 會增加每一條指令的執行時間
(C) Pipeline 容易產生 hazard
(D) 假如沒有 hazard，理論上指令可以一拍一條完成
(E) Pipeline 保證所有程式一定加速

【白話翻譯】
「哪些選項正確描述了 CPU 使用『工廠流水線（Pipeline）』方式執行指令時，會帶來的好處與壞處？」

【這題在考什麼】
你是否能分清楚 「整體產能（吞吐量）」 與 「單一工作速度（延遲）」 的差別，以及 Pipeline 先天會遇到的限制（衝突）。

正確答案揭曉

A、C、D

一句話總結：「Pipeline 提高吞吐量，但會有 hazard，也不保證每個程式都加速。」

【名師詳解】

(A) 正確： 就像生產線一樣，同時間有多個階段在處理不同的指令，單位時間內完成的指令變多了（吞吐量上升）。

(B) 錯誤（這是陷阱！）： Pipeline 不會縮短單一指令的時間。 事實上，因為要在各階段間傳遞資料（加入 latch 暫存器），單一指令從進入到離開的時間反而會稍微變長（Latency 增加）。

(C) 正確： 因為多條指令同時在管線中，前面的指令還沒寫入結果，後面的指令就要讀取，就會發生資料衝突（Data Hazard）；或者搶同一個硬體資源（Structural Hazard）。

(D) 正確： 在最理想（無衝突）的情況下，當管線填滿後，每一個時脈週期（Clock Cycle）都會有一條指令從管線末端產出。 CPI = 1。

(E) 錯誤： 如果程式充滿了分支指令（Branch）導致管線一直清空（Flush），或者衝突太多導致暫停（Stall），效能提升會很有限，不保證一定加速。

考前衝刺筆記

【記憶口訣】

「管線拚產量，不保證單快」

【常見陷阱】

  • 把「吞吐量 (Throughput)」誤當成「單一執行時間 (Latency/Execution Time)」。
  • 以為 Pipeline 是萬靈丹，實際上會有額外開銷 (Overhead)。
  • 忽略 hazard 帶來的 penalty（懲罰），導致理想 CPI 達不到 1。
