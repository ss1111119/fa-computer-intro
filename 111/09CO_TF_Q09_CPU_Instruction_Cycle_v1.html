【白話翻譯】

「CPU 做事情時，是不是先解碼（看懂），再擷取（拿來），最後才執行（去做）？」

【這題在考什麼】

「你是否記得 CPU 處理一個指令的『先後順序』。」

提示：就像你要煮菜，得先去買菜吧？

觀念教學：正確順序

CPU 是一個很聽話但很死板的工人，它做事一定嚴格遵守這個 SOP：


  • 擷取 (Fetch)
    把指令從記憶體「抓」進 CPU 的暫存器。
  • 解碼 (Decode)
    由控制單元負責翻譯，看懂這個指令到底要算數學還是搬資料。
  • 執行 (Execute)
    實際去運算或搬移資料。
生活化比喻（寫作業）：
1. 擷取：從老師那邊拿到作業單。
2. 解碼：讀題，看懂題目在問什麼。
3. 執行：動筆開始寫答案。
「沒有先拿到指令，不可能先解碼！」

選項解析

(A) O
→ 錯！ 題目說「先解碼再擷取」，這是不可能的。你手上沒有東西，要解碼什麼？
(B) X
→ 正確！ 題目敘述的順序錯誤，正確應該是擷取在最前面。

常見錯誤提醒

  • 把「解碼」和「擷取」記反： 這是最常見的錯誤，請記得「巧婦難為無米之炊」，沒有米（擷取指令）怎麼煮飯（解碼與執行）？
  • 被專有名詞嚇到： 看到 Instruction Cycle 就慌了，其實只是「拿 → 懂 → 做」而已。

記憶口訣

口訣：

「先拿、再懂、才做」

（擷取 Fetch → 解碼 Decode → 執行 Execute）

答案揭曉

X

正確答案是 (B)

一句話理由：

CPU 指令週期正確順序是「擷取 → 解碼 → 執行」，題目寫反了，所以選 X。
