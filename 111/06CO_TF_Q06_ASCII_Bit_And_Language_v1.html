ASCII 夠不夠用？先搞懂它能表示什麼

字元編碼必考陷阱題
6. ASCII 是使用 16 位元，而且足以呈現許多當今的主要文字語言。
  • (A) O
  • (B) X

請點擊上方按鈕，老師會把專有名詞翻譯成白話文給你聽！

【白話翻譯】
「ASCII 這個編碼系統是不是用 16 個 0與1 組成的？而且它的容量大到能放入現在很多國家的語言（像中文、日文、韓文等）？」

【這題在考什麼】
考你是否能分清楚：
1. ASCII（早期、美國用、英文為主）
2. Unicode（現代、全球用、各國語言）
這兩個標準的差異。

觀念教學：ASCII 到底是多少位元？

1. ASCII 的位元數（必考）
標準 ASCII 是 7 位元（7 bits），可以表示 27 = 128 個符號。
常見的「擴充 ASCII」則是使用 8 位元（1 byte），表示 256 個符號。
重點：絕對不是 16 位元。
2. ASCII 能表示哪些文字？
主要是：英文大小寫（A-Z, a-z）、阿拉伯數字（0-9）、常用標點符號與控制字元。
它不能完整表示中文、日文、韓文（CJK）這些字數成千上萬的語言。
3. 16 位元比較接近什麼？
通常提到 16 位元，指的是早期 Unicode 的實作或是 UTF-16 編碼，這才是用來解決全球語言問題的方案。

是非題判斷解析

(A) O
→ 錯。錯在兩點：
1. ASCII 不是 16 位元（是 7 或 8 位元）。
2. ASCII 容量太小，不足以表示多數語言。

(B) X
→ 正確。題目敘述把 ASCII 與 Unicode 的特色混在一起了，這是典型的張冠李戴。

常見錯誤提醒（陷阱區）

  • 看到「能表示很多語言」就直覺選 O，忘記 ASCII 其實很「小」。
  • 誤以為「位元數越大越好」，所以覺得 16 位元應該就是 ASCII（事實上 ASCII 是電腦發展早期的產物，比較精簡）。
  • 把「字元集（Unicode）」與「編碼方式（UTF-8）」搞混，但這題主要是考最基本的 ASCII 規格。
【記憶口訣】
ASCII 英文小集合（7 bit），
Unicode 全球大集合。
答案：(B) X

一句話理由：
ASCII 標準為 7 位元（擴充為 8 位元），且主要用於表示英文，空間太小，不足以涵蓋多數主要文字語言（如中文）。
